# <div>, <section>, <article> 사용법 정리

## 1. 글의 목적

HTML태그를 잘 사용하는 것도 다른 코드를 쓰는 것 만큼 중요합니다. HTML를 사용하다보면 <div>를 가장 자주 사용하게 됩니다. <div>로만 채워진 html 태그는 가독성도 떨어질뿐더러 코드 작성자의 자존감을 낮추게 됩니다.

<div>와 더불어 <section>과 <article>을 사용할 수 있습니다. 하지만 사람마다 각 태그가 의미(semantics)하는 바가 다소 다른 것 같아 각 태그의 용도와 저희 팀에서 사용 되어질 사용법 (convention)을 정리하였습니다. 

이 글을 읽으신 후 여러분은 남용되어진 <div>로 떨어진 자존감을 되찾으실 수 있습니다. 자기 자신을 사랑하는 것은 쉽지 않지만 매우 중요한 일입니다.

## 2. 태그 설명

### 2.1 div

하위 태그들을 묶을 때 사용하는 가장 보편적인 태그입니다. 보편적이지만 <div>는 아무런 의미도 가지고 있지 않습니다. 

W3C에서도 다른 더 좋은 옵션이 없을 때 사용을 추천합니다. 하지만, 다양하고 복잡한 어플리케이션을 개발하는 저희 팀에서는 ‘***사용을 피해주세요’***의 의미보다는 css 적용을 위해 class같은것들이 들어가야 하는 필요한 상황에 적절하게 사용하는 것이 좋을 것 같습니다.

***예시:***

```jsx
// 모달같은 css를 위해서 어쩔 수 없이 div가 들어가야 하는 경우
<div class="modal-container">
  <section class="modal">
    <h1>Modal Title</h1>
    <p>Text goes here</p>
  </section>
</div>

// 버튼의 위치를 잡기 위해 강제로 wrapper 같은걸 사용하기 위한 경우
<div class="button-wrapper">
	<buton class="escape-button">90층에 사람 있어요! 살려주세요!</button>
</div>
```

### 2.2 section

<section>은 <div>보다 의미가 좁지만 <nav>, <article>, <main>, <aside> 보다 보편적(general)이며 더 넓은 의미를 가지고 있습니다. 

우리가 보통 생각하는 ‘섹션(구역)’의 개념으로 생각하는게 좋습니다. 

*이 두가지 규칙을 지키면 좋을 것 같습니다:*

- <section>의 child태그들이 서로 관련되어지는 경우
    - 서로 관련이 없다면 div를 사용
- <section>보다 더 좋은 태그 이름이 생각나지 않는 경우

***예시:***

```jsx
// 이렇게 헤더를 넣어주면 좋습니다
<section class="newsletter">
  <h1>Subscribe to the Newsletter</h1>
  <form> <!-- ... --> </form>
</section>

// 우리의 상태를 하나의 섹션으로 잡고, 우리의 상태에 관련된 섹션들을 추가로 넣을 수 있기도 합니다 
<section class="our-state">
	<h2>우리의 상태</h2>
	<section class="our-state__now">
		<h3>현재는 사무실에 있습니다</h3>
		<p>지금은 사무실에서 사이좋게 일을 하고 있습니다.</p>
	</section>
	<section class="our-state__future">
		<h3>오늘 밤에 퇴근하는 미래를 그립니다</h3>
		<p>일을 끝내지 못하면 퇴근은 할 수 없습니다. 모르셨습니까?</p>
	</section>
</section>

// 😡 BAD: 이렇게 사용하지 말아주세요. <nav>같은 더 좋은 태그들이 있을지도 모르니까요!
<section class="navigation">
	<p>메뉴</p>
	<p>장바구니</p>
</section>
```

### 2.3 article

<article>은 말 그대로 ‘기사’를 뜻하는 태그입니다. 저희가 언론사였다면 자주 사용하는 태그입니다. 

분명 저희 서비스의 목적에 어울리게 쓰는 방법도 있습니다. 서비스 약관이라던지, 또는 게시글 같이 하나의 속성으로 충분히 의미가 전달 되는 경우입니다.

```jsx
<article class="post">
  <h1>오늘 업무일지</h1>
  <p>오늘은 코드를 한 줄도 안썼다.</p>
  <p>왜냐면 코드를 안 쓰고 개발을 하는 것이 제일 좋기 때문이다.</p>
  <p>개발은 미래의 내가 할 것이다.</p>
</article>
```

## **3. 요약**

***‘여러명이 작성한 코드도 한 사람이 작성한 것 처럼’*** html 태그를 작성하기 위해서, 이 부분 생각 부탁드립니다:

1. <div>는 css 구조상 class명을 짓거나 또는 flex나 grid를 사용해야 하는 상황에서 사용 - <div>는 어떠한 의미도 가지고 있지 않습니다.
2. <section>은 하위 태그들이 관련이 있을 경우 사용. 단, <section>보다 <nav>, <article>, <main> 같은 더 적합한 태그가 있다면 이를 사용 해주세요.

***MDN에서 추천하는 <section>을 사용방식입니다. 전체적으로 적용이 되는 것 같아 이 부분도 시간 되시면 읽어주세요 🙂**

- If the contents of the element represent a standalone, atomic unit of content that makes sense syndicated as a standalone piece (e.g. a blog post or blog comment, or a newspaper article), the `[<article>](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/article)` element would be a better choice.
- If the contents represent useful tangential information that works alongside the main content, but is not directly part of it (like related links, or an author bio), use an `[<aside>](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/aside)`.
- If the contents represent the main content area of a document, use `[<main>](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/main)`.
- If you are only using the element as a styling wrapper, use a `[<div>](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/div)`. A rule of thumb is that a `<section>` should logically appear in the outline of a document.
